# Распознавание рукописных символов EMNIST

Практическая работа по обучению нейронных сетей для распознавания рукописных символов с использованием датасета EMNIST Balanced и деплоем через FastAPI.

## Суть задачи

Обучить модель которая умеет распознавать рукописные символы — цифры и буквы — и развернуть её в виде веб-приложения где можно нарисовать символ и получить предсказание.

Датасет EMNIST Balanced содержит 112 800 обучающих и 18 800 тестовых изображений размером 28×28 пикселей, разбитых на 47 классов (цифры 0–9, заглавные и строчные буквы).

## Что было сделано

Работа проходила в три этапа: сначала обучили простую логистическую регрессию как базовую модель, потом MLP, и наконец CNN которая показала лучший результат.

### Логистическая регрессия

Первая модель — просто линейный слой без скрытых слоёв. Картинка разворачивается в вектор 784 признаков и подаётся напрямую в классификатор. Точность получилась около 67%, что неплохо для такой простой модели, но недостаточно для практического применения.

### MLP (многослойный перцептрон)

Вторая модель — два полносвязных слоя с активацией ReLU. Добавил BatchNorm для стабилизации обучения и Dropout для борьбы с переобучением. Оптимизатор сменил с SGD на Adam, добавили ReduceLROnPlateau scheduler который уменьшает learning rate когда модель перестаёт улучшаться.

Итоговая архитектура:

```
Linear(784 → 512) → BatchNorm → ReLU → Dropout(0.5)
Linear(512 → 256) → BatchNorm → ReLU → Dropout(0.5)
Linear(256 → 47)
```

Точность на тестовой выборке: около 86%.

### CNN (сверточная нейронная сеть)

Финальная модель. В отличие от MLP, свёрточные слои умеют работать с пространственной структурой изображения — они ищут локальные паттерны (края, кривые, углы) что важно для рукописных символов.

Архитектура:

```
Conv2d(1→32, 3×3) → ReLU → MaxPool(2×2)
Conv2d(32→64, 3×3) → ReLU → MaxPool(2×2)
Linear(1600 → 47)
```

Точность на тестовой выборке: 88%+.

## Подбор параметров

**Оптимизатор.** Начал с SGD, потом перешел на Adam — он сходится быстрее для таких задач. Learning rate 1e-3.

**Scheduler.** ReduceLROnPlateau с patience=5 — уменьшает lr в 10 раз если val_loss не улучшается 5 проверок подряд.

**Количество эпох.** 50-100 эпох, валидация каждые 10 эпох.

**Размер батча.** 64 — стандартный выбор для такого датасета.

**Аугментация при обучении.** Случайный поворот ±10° и случайный сдвиг — чтобы модель лучше обобщалась на разные стили написания.

## Предобработка входных данных

Данные из EMNIST приходят транспонированными, поэтому при инференсе нужно транспонировать входное изображение. Дальнейшая обработка:

1. Транспонирование (`.T`)
2. Утолщение линий через морфологическую дилатацию (OpenCV)
3. Сглаживание через Bilateral Filter — убирает шум сохраняя края
4. Центрирование символа через center_of_mass
5. Нормализация: деление на 255, потом стандартизация со средним 0.1307 и std 0.3081

## Запуск

### Локально

Установить зависимости:

```bash
pip install -r requirements.txt
```

Запустить сервер:

```bash
uvicorn myapp.main:app --reload
```

Приложение будет доступно по адресу http://127.0.0.1:8000

### Через Docker

Собрать образ:

```bash
docker build -t myapp_practice5 .
```

Запустить контейнер:

```bash
docker run -p 8000:8000 myapp_practice5
```

Приложение будет доступно по адресу http://localhost:8000
